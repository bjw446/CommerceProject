## 목표 : 객체 지향 설계 구조를 적용한 커머스 플랫폼
- 각 객체의 역할과 책임(R&R)을 어떻게 나눌지 고민 해보기

### [STEP 1] : 객체 지향 설계를 적용해 상품 관리 시스템을 프로그래밍
- 상품 목록 출력
- 상품 정보를 가지는 Product 클래스 생성
- 객체 생성을 통해 상품 목록 추가
- 반복문을 활용해 리스트를 탐색하여 접근

### [STEP 2] : 객체 지향 설계를 적용해 순서 제어를 클래스로 관리
- CommerceSystem 클래스 생성
- 상품 목록을 관리하는 리스트 필드 존재
- start 함수를 따로 만들어 관리

### [STEP 3] : 객체 지향 설계를 적용해 카테고리와 고객 클래스로 관리
- 각 카테고리 내에 상품 생성 및 상품 목록 관리
- 카테고리 별 메서드 구현 
- 고객 정보를 관리하는 클래스 생성

### [STEP 4] : 캡슐화 적용하기
- Getter, Setter 메서드를 사용해 데이터 관리하기

### [STEP 5] : 장바구니 및 주문하기 기능 추가
- 장바구니 생성 및 관리 기능
- 장바구니 출력 및 금액 계산
- 장바구니 담기 및 주문 기능
- 재고 관리 시스템, 주문 완료시 재고 변경

### [STEP 6] : 관리자 모드 추가
- 관리자 인증을 통한 로그인 기능
- 신규 상품 추가 및 수정, 삭제 기능
- 기존 데이터를 탐색하여 중복 및 재고 관리

### [STEP 7] : Enum, 람다 & 스트림을 활용한 관리
- Enum을 활용한 고객 등급별 할인율 관리
- 람다 & 스트림을 활용한 상품 검색 및 관리
